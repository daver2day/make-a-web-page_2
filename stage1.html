Web Development Important Concepts

Lesson 1: Concepts of the Web

HTTP - How the web works

The web consists of computers that talk to each other over the Internet. The computers talk to each other with a protocol called http. The end user's computer runs a web browser which is responsible for requesting documents from servers, which store them, and displaying the contents to the user. The user's computer sends a document request to the server using the http protocol. When the server receives the request, it looks for the document and returns it to the user also using the http protocol. A good example video of how this works can be found here: This video explains the process.

HTML

The primary document type that makes up the web is called a HTML document. HTML stands for Hypertext Markup Language and is used to control the way the document is displayed to the user. The user's computer reads the document and displays it as described by the HTML code. The HTML document consists of content to be displayed as well as instructions on how to display the content. A good overview of HTML documents can be seen in this video.

Tags and Elements

HTML documents consists of various parts, but they primarily consist of instructions on how to format the content, called tags and the content itself. This combination of tags and content is often referred to as an Element. A tag is used to tell the browser how to treat the contents as well as providing a means to link to external content not contained in the document itself. Tags and elements are explained in more detail in this video. Referring to external content is a powerful way to create interesting and dynamic pages that can display pictures, videos, or play music.

Computers are stupid

Computers only follow instructions as provided to them. They interpret the instructions literally, so even very small mistakes in those instructions can cause unexpected behavior.

In-line vs Block Elements

HTML elements have two primary distinguishing characteristics that allow certain degrees of control. An In-line element is basic item that is usually rendered similar to ordinary text. A Block element, however, forms a sort of invisible block around the content such that it can be formatted in various ways.

Lesson 2: Structure of HTML pages and the tools to create and inspect them

Browser DOM - Tree Structure

Although we've seen that HTML pages are composed of a series of elements which, in turn, are composed of tags and content, there is a bigger picture to the overall structure of HTML Pages.

When the browser processes the HTML page content, it translates it into a tree like structure. This tree like structure is controlled or defined by the DOM, or Document Object Model. This provides a consistent way for browsers to understand the structural and style components of the page.

Developer tools

A very useful feature of modern browsers is provided by developer tools within the browser. The developer tools allow you to view the overall tree structure of the page in a tabbed view while also separating the style elements to make it easier to view the overall page design.

So many Boxes

In order to bring the overall design of HTML pages into an easily manageable entity, they can be viewed, in a larger context, as a series of rectangular boxes. Doing this allows the page designer a great deal of flexibility in arranging the overall look of the page.

Style of HTML Pages

Having a structure to our HTML page, it largely would look pretty bland unless we had a method to add character to the page design. The overall style of HTML pages is controlled by the CSS language.

CSS Style

CSS provides features to control the visual styling of the page such as header sizes, colors, font types and sizes, border sizes, and box placement among many others. Besides setting style characteristics for HTML pages, CSS also provides for setting style parameters in a consistent manner across the entire page. This reduces the need to duplicate style parameters for each page element.

HTML page design tools

Various code creation tools are available to the page designer. Each offers different features ranging from very basic to more advanced.

Scratchpad

Scratchpad is a very simplistic code creation tool. It provides for immediate feedback on page rendering, but only supports the HTML and CSS languages and does not provide for page sharing. Scratchpad is very good for beginners to experiment with HTML and CSS concepts.

Codepen

Codepen is a step up from Scratchpad as it offers the addition of Java-script support, but at the cost of slightly slower page rendering feedback. It also provides support for page sharing.

Sublime Text

Sublime Text is a multi-featured text editor tool that supports many page design languages, but is more difficult to learn and use, but offers very advanced design capabilities. Page rendering is done only on page refresh and sharing is not possible with Sublime. It is geared towards the more advanced developer.

HTML Code Validation

Ensuring that Web pages can be properly displayed in the majority of browsers is important to enable the widest possible audience to view and interact with the page. However, HTML page code can get very complicated and hard for for the web page developer to catch all the possible errors. While the browser developer tools are very helpful, another tool can be used to analyze and display coding mistakes. One such tool is the W3C Markup Validation Service found here: W3C Markup Validation Service. In addition to finding HTML coding errors, this service also provides suggestion on how to write code to be compatible with a very large number of browsers in use.

Lesson 3: Using CSS code to improve Web page appearance and reduce coding errors

Using CSS code to improve Web page appearance while reducing errors.

While it is useful to think of HTML pages as a collection of boxes, there often are so many pieces to a page that trying to position all of them can be tedious. In addition, all those boxes contain content and both the boxes and content need to look good too. If the Web page developer treated each box and every piece of content individually, while trying to adjust the visual attributes, it would quickly become a nightmare to get a consistent look while also avoiding errors.

A great way to get this consistent look while avoiding errors, would be to use the CSS language. CSS stands for Cascading Style Sheets. Cascading refers to the ability to allow for style properties to be inherited from other properties which reduces the chance of creating an inconsistent look.

While CSS instructions can be contained within the HTML page header, it is often a much better idea to keep the CSS code in a separate file. This CSS file is known as a CSS style sheet. Placing the CSS code in a separate file allows it to be kept separate from the HTML code, but also allows for better reuse across multiple web pages. This reduces the likelihood of errors sneaking into the code.

Another very significant benefit to using CSS code, is that provides a method to reduce the repetition of code entries. By using CSS, visual attributes can be set and applied once rather than entering them again each time they are needed. This reduces code complexity since attributes only need to be entered once, but also reduces the chances for making typo errors or missing attributes on some items.

CSS Code Validation

As discussed preciously, it is important to have Web pages display properly in as many browsers as possible. In addition to the W3C Markup Validation Service mentioned above, there is a corresponding service to validate the CSS code used to control Web page style. The W3C Organization provides a tool to check CSS code and is located at: W3C CSS Validation Service.
